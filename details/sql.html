Situation (Background)

Employee payroll systems within medium to large-scale corporate environments demand a high degree of precision, strict data integrity, and optimal financial data security. Manual recapitulation processes for basic salary components, bonuses, overtime calculations, allowances, up to the accumulation of mandatory deductions often slow down financial department operations and pose risks of financial data inconsistency. To address these challenges, this project aims to build a relational database architecture foundation named perusahaan using MySQL that is capable of automating the entire payroll business workflow in a synchronized manner.

Task (Challenges & Responsibilities)

My primary responsibility in this project was to design an optimal relational table schema and implement data normalization to prevent manipulation anomalies. The greatest technical challenge was integrating multiple dynamic financial components (such as overtime, allowances, and deductions tables) into a single final compilation table (gaji) with strict dependency rules. Furthermore, I was required to shift the burden of data update business logic from the application layer (application level) directly into the database server using Stored Procedures and Database Triggers to ensure query efficiency and data transaction security.

Action (Technical Implementation & Query Structure)

The design of this database architecture was realized by leverage advanced operational features of MySQL Enterprise. The tactical steps implemented include:

1. Comprehensive Relational Schema Design

Building a relational database structure consisting of 9 interconnected operational entities using Foreign Key constraints with ON DELETE CASCADE and ON UPDATE CASCADE rules to guarantee referential integrity:

  • Profile Entities: The Karyawan table as the central entity, supported by the Jabatan and Departemen tables to structure the organizational hierarchy.
  • Operational & Attendance Entities: The Absensi table to track attendance and the Lembur table to calculate employee overtime hours.
  • Financial Component Entities: The Tunjangan, TunjanganKaryawan, and PotonganGaji tables to store salary addition and deduction variables.
  • Final Recapitulation Entity: The Gaji table which compiles the total net pay along with its regular payment dates.
2. Business Logic Automation via Stored Procedures

To simplify application interaction with the database and maintain code modularity, I designed several advanced stored procedures with dynamic input parameters:

  • Collective Data Insertion Procedure (InsertDataLengkap): Automates adding new data sequentially across multiple tables in a single command execution (Karyawan, Lembur, Potongan, Tunjangan, and initial Gaji).
  • Centralized Update Procedure (UpdateDataLengkap): Provides a single secure gateway to update all employee financial parameters simultaneously across related relational tables, ensuring backend program code efficiency.
3. Securing Transactions via SQL Dump Configuration

Structuring a secure SQL dump script configuration by disabling index keys verification (INDEX_KEYS=0) and foreign key constraints (FOREIGN_KEY_CHECKS=0) during the data migration process to accelerate data importing, and then automatically re-enabling them to maintain ACID (Atomicity, Consistency, Isolation, Durability) validity.

Result (Final Outcome & Impact)

Through this structured relational database design, the corporate payroll management system successfully operates with high performance stability. The utilization of Stored Procedures has proven to eliminate repetitive query latency from the web backend side and unifies data manipulation validation directly inside the database server. This normalized data structure successfully prevents data duplication, accelerates monthly payroll report processing speeds, and significantly minimizes the risk of employee financial calculation errors.
